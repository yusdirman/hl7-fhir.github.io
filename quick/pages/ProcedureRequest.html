ProcedureRequest

  • A request for a procedure to be performed

    Profile of ProcedureRequest for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        approachBodySite 0..* List<BodySite>
      The requested approach body site used for this procedure. Multiple locations are allowed.
        appropriatenessScore[x] 0..1 code | Decimal
      The RAND appropriateness score for the requested procedure.

      Binding: QICoreAppropriatenessScore (Required)
        asNeeded[x] 0..1 Boolean | Concept
      If a CodeableConcept is present, it indicates the pre-condition for performing the procedure.

      (Example)
        bodySite 0..* List<Concept>
      Indicates the sites on the subject's body where the procedure should be performed (I.e. the target sites).

      Binding: SNOMED CT Body Structures (Example)
        code 1..1 Concept
      The specific procedure that is ordered. Use text if the exact nature of the procedure cannot be coded.

      Binding: DAFProcedureType (Preferred)
        encounter 0..1 Encounter
      The encounter within which the procedure proposal or request was created.
        identifier 0..* List<Identifier>
      Identifiers assigned to this order by the order or by the receiver.
        notes 0..* List<Annotation>
      Any other notes associated with this proposal or order - e.g. provider instructions.
        orderedOn 0..1 DateTime
      The time when the request was made.
        orderer 0..1 Practitioner | Patient | RelatedPerson | Device
      The healthcare professional responsible for proposing or ordering the procedure.
        performer 0..1 Practitioner | Organization | Patient | RelatedPerson
      For example, the surgeon, anaethetist, endoscopist, etc.
        priority 0..1 code
      The clinical priority associated with this order.

      Binding: ProcedureRequestPriority (Required)
        reasonRefused 0..1 Concept
      The reason the request was refused. Applies only if the status is refused.

      Binding: Procedure Not Performed Reason (SNOMED-CT) (Example)
        reason[x] 0..1 Concept | Condition
      The reason why the procedure is being proposed or ordered. This procedure request may be motivated by a Condition for instance.

      Binding: Procedure Reason Codes (Example)
        scheduled[x] 0..1 DateTime | interval<DateTime> | Timing
      The timing schedule for the proposed or ordered procedure. The Schedule data type allows many different expressions. E.g. "Every 8 hours"; "Three times a day"; "1/2 an hour before breakfast for 10 days from 23-Dec 2011:"; "15 Oct 2013, 17 Oct 2013 and 1 Nov 2013".
        status 0..1 code
      The status of the order.

      Binding: ProcedureRequestStatus (Required)
        subject 1..1 Patient | Group
      The person, animal or group that should receive the procedure.
