Attachment

  • Content in a format defined elsewhere

    Base StructureDefinition for Attachment Type

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        contentType 0..1 code
      Identifies the type of the data in the attachment and allows a method to be chosen to interpret or render the data. Includes mime type parameters such as charset where appropriate.

      Binding: value set (Required)
        creation 0..1 DateTime
      The date that the attachment was first created.
        data 0..1 base64Binary
      The actual data of the attachment - a sequence of bytes. In XML, represented using base64.
        hash 0..1 base64Binary
      The calculated hash of the data using SHA-1. Represented using base64.
        size 0..1 unsignedInt
      The number of bytes of data that make up this attachment.
        title 0..1 String
      A label or set of text to display in place of the data.
        url 0..1 Uri
      An alternative location where the data can be accessed.
