DiagnosticOrder

  • A request for a diagnostic service

    Profile of DiagnosticOrder for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        encounter 0..1 Encounter
      An encounter that provides additional information about the healthcare context in which this request is made.
        event 0..* List<Event> (TBD)
      A summary of the events of interest that have occurred as the request is processed; e.g. when the order was made, various processing steps (specimens received), when it was completed.
        identifier 0..* List<Identifier>
      Identifiers assigned to this order instance by the orderer and/or the receiver and/or order fulfiller.
        item 0..* List<Item> (TBD)
      The specific diagnostic investigations that are requested as part of this request. Sometimes, there can only be one item per request, but in most contexts, more than one investigation can be requested.
        note 0..* List<Annotation>
      Any other notes associated with this patient, specimen or order (e.g. "patient hates needles").
        orderer 0..1 Practitioner
      The practitioner that holds legal responsibility for ordering the investigation.
        priority 0..1 code
      The clinical priority associated with this order.

      Binding: DiagnosticOrderPriority (Required)
        reason 0..* List<Concept>
      An explanation or justification for why this diagnostic investigation is being requested. This is often for billing purposes. May relate to the resources referred to in supportingInformation.

      Binding: Problem Value Set (Preferred)
        reasonRejected 0..1 Concept
      The coded reason why the diagnostic order was rejected. The element must be used only when status is rejected.

      Binding: Reasons for rejecting a diagnostic order (Example)
        specimen 0..* List<Specimen>
      One or more specimens that the diagnostic investigation is about.
        status 0..1 code
      The status of the order.

      Binding: DiagnosticOrderStatus (Required)
        subject 1..1 Patient | Group | Location | Device
      Who or what the investigation is to be performed on. This is usually a human patient, but diagnostic tests can also be requested on animals, groups of humans or animals, devices such as dialysis machines, or even locations (typically for environmental scans).
        supportingInformation 0..* List<Observation | Condition | DocumentReference>
      Additional clinical information about the patient or specimen that may influence test interpretations. This includes observations explicitly requested by the producer(filler) to provide context or supporting information needed to complete the order.
