DiagnosticReport

  • A Diagnostic report - a combination of request information, atomic results, images, interpretation, as well as formatted reports

    Profile of DiagnosticReport for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        category 0..1 Concept
      A code that classifies the clinical discipline, department or diagnostic service that created the report (e.g. cardiology, biochemistry, hematology, MRI). This is used for searching, sorting and display purposes.

      Binding: Diagnostic Service Section Codes (Example)
        code 1..1 Concept
      A code or name that describes this diagnostic report.

      Binding: US Laboratory Observation Profile Observation Name Codes (Preferred)
        codedDiagnosis 0..* List<Concept>
      Codes for the conclusion.

      Binding: SNOMED CT Clinical Findings (Example)
        conclusion 0..1 String
      Concise and clinically contextualized narrative interpretation of the diagnostic report.
        effective[x] 1..1 DateTime | interval&lt;DateTime&gt;
      The time or time-period the observed values are related to. When the subject of the report is a patient, this is usually either the time of the procedure or of specimen collection(s), but very often the source of the date/time is not known, only the date/time itself.
        encounter 0..1 Encounter
      The link to the health care event (encounter) when the order was made.
        identifier 0..* List<Identifier>
      The local ID assigned to the report by the order filler, usually by the Information System of the diagnostic service provider.
        image 0..* List<Image> (TBD)
      A list of key images associated with this report. The images are generally created during the diagnostic process, and may be directly of the patient, or of treated specimens (i.e. slides of interest).
        imagingStudy 0..* List<ImagingStudy | ImagingObjectSelection>
      One or more links to full details of any imaging performed during the diagnostic investigation. Typically, this is imaging performed by DICOM enabled modalities, but this is not required. A fully enabled PACS viewer can use this information to provide views of the source images.
        issued 1..1 DateTime
      The date and time that this version of the report was released from the source diagnostic service.
        locationPerformed 0..1 Location
      Facility location where this report was prepared.
        performer 1..1 Practitioner | Organization
      The diagnostic service that is responsible for issuing the report.
        presentedForm 0..* List<Attachment>
      Rich text representation of the entire result as issued by the diagnostic service. Multiple formats are allowed but they SHALL be semantically equivalent.
        request 0..* List<DiagnosticOrder | ProcedureRequest | ReferralRequest>
      Details concerning a test or procedure requested.
        result 0..* List<Observation>
      Observations that are part of this diagnostic report. Observations can be simple name/value pairs (e.g. "atomic" results), or they can be grouping observations that include references to other members of the group (e.g. "panels").
        specimen 0..* List<Specimen>
      Details about the specimens on which this diagnostic report is based.
        status 1..1 code
      The status of the diagnostic report as a whole.

      Binding: DiagnosticReportStatus (Required)
        subject 1..1 Patient | Group | Device | Location
      The subject of the report. Usually, but not always, this is a patient. However diagnostic services also perform analyses on specimens collected from a variety of other sources.
