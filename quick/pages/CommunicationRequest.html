CommunicationRequest

  • A request for information to be sent to a receiver

    Profile of CommunicationRequest for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        category 0..1 Concept
      The type of message to be sent such as alert, notification, reminder, instruction, etc.

      (Example)
        encounter 0..1 Encounter
      The encounter within which the communication request was created.
        identifier 0..* List<Identifier>
      A unique ID of this request for reference purposes. It must be provided if user wants it returned as part of any output, otherwise it will be autogenerated, if needed, by CDS system. Does not need to be the actual ID of the source system.
        medium 0..* List<Concept>
      A channel that was used for this communication (e.g. email, fax).

      Binding: QICore Communication Medium Codes (Example)
        payload 0..* List<Payload> (TBD)
      Text, attachment(s), or resource(s) to be communicated to the recipient.
        priority 0..1 Concept
      Characterizes how quickly the proposed act must be initiated. Includes concepts such as stat, urgent, routine.

      Binding: DiagnosticOrderPriority (Example)
        reason 0..* List<Concept>
      The reason or justification for the communication request.

      (Example)
        reasonRejected 0..1 Concept
      The reason associated with the rejection of this communication request by the receiver. Applicable only if the status is rejected.

      Binding: Communication reason not performed codes (Example)
        recipient 0..* List<Practitioner | Patient | Device | RelatedPerson | Organization>
      The entity (e.g. person, organization, clinical information system, or device) which is the intended target of the communication.
        requestedOn 0..1 DateTime
      The time when the request was made.
        requester 0..1 Patient | Practitioner | RelatedPerson
      The responsible person who authorizes this order, e.g. physician. This may be different than the author of the order statement, e.g. clerk, who may have entered the statement into the order entry application.
        scheduled[x] 0..1 DateTime | interval&lt;DateTime&gt;
      The time when this communication is to occur.
        sender 0..1 Practitioner | Patient | Device | RelatedPerson | Organization
      The entity (e.g. person, organization, clinical information system, or device) which is to be the source of the communication.
        status 0..1 code
      The status of the proposal or order.

      Binding: CommunicationRequestStatus (Required)
        subject 0..1 Patient
      The patient who is the focus of this communication request.
