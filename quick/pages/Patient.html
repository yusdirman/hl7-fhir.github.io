Patient

  • Information about an individual or animal receiving health care services

    Profile of Patient for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        NCT 1..1 String
      The National Clinical Trial number. The format for the US ClinicalTrials.gov registry number is "NCT" followed by an 8-digit number, e.g.: NCT00000419.
        active 0..1 Boolean
      Whether this patient record is in active use.
        address 0..* List<Address>
      Addresses for the individual.
        birthDate 0..1 DateTime
      The date of birth for the individual.
        birthPlace 0..1 Address
      The registered place of birth of the patient. A sytem may use the address.text if they don't store the birthPlace address in discrete elements.
        birthTime 0..1 DateTime
      The time of day that the Patient was born. This includes the date to ensure that the timezone information can be communicated effectively.
        cadavericDonor 0..1 Boolean
      Flag indicating whether the patient authorized the donation of body parts after death.
        careProvider 0..* List<Organization | Practitioner>
      Patient's nominated care provider.
        clinicalTrial 0..* List<Element>
      The clinical trials this patient has or is participating in.
        code 0..1 Extension
      Code representing nationality of patient.
        communication 0..* List<Communication> (TBD)
      Languages which may be used to communicate with the patient about his or her health.
        contact 0..* List<Contact> (TBD)
      A contact party (e.g. guardian, partner, friend) for the patient.
        deceased[x] 0..1 Boolean | DateTime
      Indicates if the individual is deceased or not.
        disability 0..* List<Concept>
      Value(s) identifying physical or mental condition(s) that limits a person's movements, senses, or activities.

      Binding: Disability Value Set (Preferred)
        ethnicity 0..1 Concept
      A code classifying the person into a named category of humans sharing a common real or presumed heritage.

      Binding: Ethnicity group (Preferred)
        extension.extension 1..1 Concept
      Code representing nationality of patient.

      Binding: value set (Required)
        gender 0..1 code
      Administrative Gender - the gender that the patient is considered to have for administration and record keeping purposes.

      Binding: AdministrativeGender (Required)
        identifier 0..* List<Identifier>
      An identifier for this patient.
        link 0..* List<Link> (TBD)
      Link to another patient resource that concerns the same actual patient.
        managingOrganization 0..1 Organization
      Organization that is the custodian of the patient record.
        maritalStatus 0..1 Concept
      This field contains a patient's most recent marital (civil) status.

      Binding: Marital Status Codes (Required)
        militaryService 0..1 Concept
      Status indicating the current or former military service of the patient.

      Binding: Military Service Codes (Example)
        multipleBirth[x] 0..1 Boolean | Integer
      Indicates whether the patient is part of a multiple or indicates the actual birth order.
        name 0..* List<HumanName>
      A name associated with the individual.
        nationality 0..* List<Element>
      The nationality of the subject.
        period 0..1 Extension
      Period when nationality was effective.
        photo 0..* List<Attachment>
      Image of the patient.
        race 0..1 Concept
      A code classifying the person into a named category of humans sharing common history, traits, geographical origin or nationality.

      Binding: MU Race Value Set (Preferred)
        reason 0..1 Concept
      The indication or reason the patient is part of this trial.

      Binding: Problem Value Set (Preferred)
        religion 0..1 Concept
      A code classifying a person's professed religion.

      Binding: v3 Code System ReligiousAffiliation (Extensible)
        telecom 0..* List<ContactPoint>
      A contact detail (e.g. a telephone number or an email address) by which the individual may be contacted.
