MedicationDispense

  • Dispensing a medication to a named patient

    Profile of MedicationDispense for decision support/quality metrics. Defines the core set of elements and extensions for quality rule and measure authors.

    • Key:  = Must support,  = QICore-defined extension,  = Is-Modifier

       Field Card. Type and Description
        authorizingPrescription 0..* List<MedicationOrder>
      Indicates the medication order that is being dispensed against.
        daysSupply 0..1 Quantity
      The amount of medication expressed as a timing amount.
        destination 0..1 Location
      Identification of the facility/location where the medication was shipped to, as part of the dispense event.
        dispenser 0..1 Practitioner
      The individual responsible for dispensing the medication.
        dosageInstruction 0..* List<DosageInstruction> (TBD)
      Indicates how the medication is to be used by the patient.
        identifier 0..1 Identifier
      Identifier assigned by the dispensing facility - this is an identifier assigned outside FHIR.
        medication[x] 1..1 Concept | Medication
      Identifies the medication being administered. This is either a link to a resource representing the details of the medication or a simple attribute carrying a code that identifies the medication from a known list of medications.

      Binding: Medication Clinical Drug (RxNorm) (Preferred)
        note 0..1 String
      Extra information about the dispense that could not be conveyed in the other attributes.
        patient 0..1 Patient
      A link to a resource representing the person to whom the medication will be given.
        quantity 0..1 Quantity
      The amount of medication that has been dispensed. Includes unit of measure.
        receiver 0..* List<Patient | Practitioner>
      Identifies the person who picked up the medication. This will usually be a patient or their caregiver, but some cases exist where it can be a healthcare professional.
        refillsRemaining 0..1 Integer
      Number of refills allowed or remaining after a dispensing event.
        status 0..1 code
      A code specifying the state of the set of dispense events.

      Binding: MedicationDispenseStatus (Required)
        substitution 0..1 Substitution (TBD)
      Indicates whether or not substitution was made as part of the dispense. In some cases substitution will be expected but does not happen, in other cases substitution is not expected but does happen. This block explains what substitution did or did not happen and why.
        type 0..1 Concept
      Indicates the type of dispensing event that is performed. For example, Trial Fill, Completion of Trial, Partial Fill, Emergency Fill, Samples, etc.

      Binding: ActPharmacySupplyType (Example)
        validityPeriod 0..1 interval&lt;DateTime&gt;
      The time interval when the medication must be administered. The start and end times must be expressed as dateTimes, not times relative to the dispensing time (e.g. use within 24 hours).
        whenHandedOver 0..1 DateTime
      The time the dispensed product was provided to the patient or their representative.
        whenPrepared 0..1 DateTime
      The time when the dispensed product was packaged and reviewed.
